      this backend will be used in a website or mobile application so you need to set the Permission for the models

      Post /application/{app_id}/ai_update_structure_permission endpoint ,expects body request: [
          {
              model: model_id,
              group: string (visitors, users),
              access_type: string (normal_access, full_control, no_access),
              limited_data_access: bool
          },
          {
              model: model_id,
              group: string (visitors, users),
              access_type: string (normal_access, full_control, no_access),
              limited_data_access: bool
          },
      ]

      mode: the target model id
      group: "users" that's mean after the user login , "visitors" before login
      access_type: 
      "normal_access": (see data only) lets say just a normal visitor to the website for example if the website is an e-commerce website he should be able to see the products data from the product model so you give normal_access
      "full_control": if you want to give full access and create update and delete the data like admin
      "no_access": the user will have no access to see or update or create or delete data
      "limited_data_access": limit access to only your own data for example if we are talking about orders model so the user can have access to this model but also he can only see and update his own data so set this to true
